CSV Diff 비교기

두 CSV 파일을 붙여넣으면 추가, 삭제, 변경된 행을 색상으로 구분하여 테이블 형태로 비교합니다. 데이터 마이그레이션, 버전 비교, 데이터 품질 검사에 유용합니다.

자주 묻는 질문

CSV Diff는 어디에 활용하나요?

데이터 마이그레이션 검증, 스프레드시트 버전 비교, 데이터 품질 감사, API 응답 데이터 비교에 사용합니다.

구분자를 변경할 수 있나요?

예, 쉼표(,), 탭(⇥), 세미콜론(;)을 지원합니다.

행 비교 기준은 무엇인가요?

첫 번째 열을 키로 사용하여 행을 매칭합니다.

대용량 CSV도 지원하나요?

브라우저 내에서 처리하므로 수천 행까지 지원하나, 매우 큰 파일은 느릴 수 있습니다.

헤더 없는 CSV도 비교 가능한가요?

예, "첫 행이 헤더" 옵션을 해제하면 헤더 없이 비교합니다.

셀 내 쉼표가 있는 경우는?

따옴표로 감싸진 필드(예: "값,포함")는 올바르게 파싱됩니다.

엑셀 파일도 비교할 수 있나요?

엑셀 파일을 CSV로 저장한 후 붙여넣기 하세요.

결과를 CSV로 내보낼 수 있나요?

현재 직접 내보내기는 지원하지 않습니다. 결과를 복사하여 사용하세요.

키 컬럼 기준 비교와 단순 행 순서 비교는 어떻게 다른가요?

이 도구는 기본적으로 첫 번째 열을 키로 사용해 행을 매칭하는 키 기준 비교를 합니다. 예를 들어 CSV A에 id=1,2,3 순서로 있고 CSV B에 id=2,1,3처럼 순서만 바뀌어 있어도, 키 기준 비교는 id로 각 행을 찾아 매칭하므로 순서 변경은 diff로 표시되지 않고 실제 값이 바뀐 셀만 표시됩니다. 반대로 단순 행 순서 비교(줄 번호 기준)를 사용하면 정렬만 다른 두 CSV도 모든 행이 "변경됨"으로 잘못 표시되는 문제가 생깁니다. 정렬이 다를 수 있는 데이터는 반드시 키 기준 비교를 사용하세요.

인코딩·구분자 문제로 diff가 깨지는 경우는?

① 한국 엑셀에서 CSV 저장 시 기본 인코딩이 EUC-KR(또는 CP949)인 경우가 많은데, UTF-8로 인식하면 한글이 깨져 모든 행이 다르게 표시됩니다. ② UTF-8 BOM(Byte Order Mark) 유무 차이 — BOM이 있는 파일과 없는 파일을 비교하면 첫 번째 셀 값 앞에 보이지 않는 문자가 붙어 첫 열이 항상 다르게 표시되는 흔한 오류가 발생합니다. ③ 구분자 차이 — 유럽/한국 로케일의 엑셀은 세미콜론(;)을, 미국 로케일은 쉼표(,)를 기본 구분자로 사용해 두 CSV의 구분자가 다르면 파싱 자체가 실패합니다. 비교 전 두 파일의 인코딩과 구분자를 반드시 통일하세요.

엑셀 자체의 비교 기능과 이 도구는 무엇이 다른가요?

엑셀(스프레드시트 비교 도구 포함)은 셀 서식(색상, 폰트, 테두리)까지 비교 대상에 포함하는 경우가 많아, 값은 같아도 서식이 다르면 차이로 표시될 수 있습니다. 이 도구는 순수하게 셀의 텍스트 값만 비교하므로, 데이터 마이그레이션 검증처럼 "값이 실제로 같은가"만 확인하고 싶을 때 더 정확하고 노이즈가 적습니다.
