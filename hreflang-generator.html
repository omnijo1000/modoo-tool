hreflang이란?

hreflang은 구글에게 동일한 콘텐츠의 다국어 또는 지역별 버전이 있음을 알려주는 HTML link 태그 속성입니다. 올바른 언어 버전의 페이지를 해당 언어 사용자에게 검색 결과로 보여주며, 중복 콘텐츠 문제를 방지합니다. 국제 SEO의 핵심 요소입니다.

자주 묻는 질문

hreflang이란 무엇인가요?

hreflang은 HTML head 또는 XML 사이트맵에 추가하는 태그로, 구글에게 페이지의 언어 및 지역 버전을 알립니다. 예를 들어 영어 사용자에게는 /en/ 페이지를, 한국어 사용자에게는 /ko/ 페이지를 검색 결과에 표시하도록 안내합니다.

x-default는 무엇을 의미하나요?

x-default는 지정된 언어와 일치하지 않는 사용자에게 표시할 기본 페이지를 지정합니다. 예를 들어 한국어, 영어, 중국어만 있고 프랑스어 사용자가 방문하면 x-default URL이 표시됩니다. 일반적으로 언어 선택 페이지나 기본 언어 페이지에 사용합니다.

언어 코드 vs 지역 코드 차이는?

언어만 지정: "en"은 모든 영어 사용자에게 적용됩니다. 언어+지역 지정: "en-US"는 미국 영어, "en-GB"는 영국 영어 사용자에게만 적용됩니다. 지역별로 다른 콘텐츠(가격, 단위 등)가 있을 때 언어+지역 코드를 사용합니다.

잘못된 hreflang 설정 시 문제는?

잘못된 hreflang은 구글이 무시합니다. 순위 하락 패널티는 없지만 의도한 언어 타겟팅 효과를 얻지 못합니다. 양방향 참조가 없거나 URL 오타, 자기 참조 누락 등이 흔한 오류입니다.

양방향 참조가 필요한 이유는?

페이지 A에서 페이지 B를 hreflang으로 가리키면, 페이지 B도 페이지 A를 가리켜야 합니다. 이를 양방향 참조라고 하며, 구글이 hreflang 설정을 신뢰하는 데 필수입니다. 단방향만 있으면 구글이 무시할 수 있습니다.

모든 페이지에 hreflang을 추가해야 하나요?

다국어 버전이 있는 모든 페이지에 추가해야 합니다. head 태그, XML 사이트맵, HTTP 응답 헤더 중 하나의 방법으로 구현할 수 있습니다. 사이트맵 방식은 동적으로 페이지 수가 많을 때 관리가 쉽습니다.

사이트맵 vs head 태그 중 어느 방법이 낫나요?

페이지 수가 적으면 head 태그가 직관적입니다. 수백~수천 페이지라면 XML 사이트맵이 중앙 관리에 유리합니다. HTTP 헤더 방식은 PDF 등 비HTML 파일에 사용합니다. 구글은 세 가지 방법 모두 동등하게 처리합니다.

zh-CN과 zh의 차이는?

"zh"는 언어만 지정하는 반면 "zh-CN"은 중국 본토, "zh-TW"는 대만, "zh-HK"는 홍콩 지역을 지정합니다. 간체자(중국 본토)와 번체자(대만·홍콩)로 나뉘기 때문에 실제 콘텐츠가 다르다면 지역 코드까지 포함하는 것이 좋습니다.
