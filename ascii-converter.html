ASCII 변환기란?

텍스트를 10진수, 16진수, 2진수, 8진수 ASCII 코드로 변환하거나 반대로 ASCII 코드를 텍스트로 복원하는 무료 온라인 도구입니다. ASCII(American Standard Code for Information Interchange)는 1963년 제정된 문자 인코딩 표준으로, 0~127의 숫자에 영문 대소문자·숫자·특수문자·제어 문자를 대응시킵니다. 예를 들어 'A'는 65, 'a'는 97, '0'은 48, 스페이스는 32입니다. 이 변환기는 프로그래밍 디버깅, 네트워크 패킷 분석, 데이터 인코딩 학습, 컴퓨터 과학 교육 등에 활용됩니다. 모든 변환은 브라우저에서 즉시 처리되며 데이터가 서버로 전송되지 않습니다.

자주 묻는 질문

ASCII 코드란?

0~127의 숫자로 문자를 표현하는 표준입니다. 'A'=65, 'a'=97, '0'=48, 스페이스=32입니다.

ASCII와 유니코드의 차이는?

ASCII는 128자, 유니코드는 110만 개 이상의 문자를 지원합니다. ASCII는 유니코드의 부분집합입니다.

한글은 ASCII로 변환되나요?

한글은 ASCII 범위(0-127)를 벗어나므로 직접 변환되지 않습니다. 유니코드 변환기를 사용하세요.

16진수(Hex) 모드는 어디서 쓰나요?

색상 코드, 메모리 주소, 네트워크 패킷 분석 등에 사용합니다.

2진수(Binary) 변환은?

컴퓨터 내부 표현을 학습하거나 비트 조작을 이해할 때 유용합니다.

쉼표 구분이란?

출력을 "72,101,108" 형태로 쉼표로 구분합니다. 기본은 공백 구분입니다.

ASCII 코드표도 제공하나요?

ASCII 테이블 도구에서 0~127 전체 ASCII 코드표를 확인할 수 있습니다.

제어 문자(LF, CR)도 변환되나요?

네, 줄바꿈(10), 캐리지리턴(13), 탭(9) 등 제어 문자도 변환됩니다. 제어 문자는 ASCII 0~31과 127에 해당하는 화면에 표시되지 않는 문자로, 통신 제어와 텍스트 서식 지정에 사용됩니다.

ASCII 코드의 범위는 어떻게 되나요?

표준 ASCII는 0~127(7비트)입니다. 0~31은 제어 문자, 32~126은 출력 가능 문자(스페이스, 숫자, 영문, 특수문자), 127은 DEL입니다. 128~255는 확장 ASCII라 불리지만 표준이 아니며 코드 페이지마다 문자가 다릅니다.

스페이스와 줄바꿈의 ASCII 코드는?

스페이스는 32(16진수 0x20), 줄바꿈(LF)은 10(0x0A), 캐리지리턴(CR)은 13(0x0D), 탭은 9(0x09)입니다. Windows는 줄바꿈에 CR+LF(13 10)를, Unix·Linux·macOS는 LF(10)만 사용합니다.

ASCII는 지금도 사용되나요?

네. HTTP 헤더, 이메일 프로토콜(SMTP), URL, 프로그래밍 언어 소스 코드, JSON 키 등 대부분의 인터넷 프로토콜과 데이터 형식이 ASCII 기반입니다. UTF-8이 ASCII와 하위 호환되도록 설계되어 ASCII 텍스트는 그대로 유효한 UTF-8입니다.

ASCII와 UTF-8의 차이는?

UTF-8은 유니코드 전체를 1~4바이트 가변 길이로 인코딩하는 방식입니다. 0~127 범위에서는 UTF-8과 ASCII가 바이트 단위로 완전히 동일하므로 모든 ASCII 파일은 그대로 유효한 UTF-8 파일입니다. 한글·이모지 등은 UTF-8에서 2~4바이트를 사용합니다.
