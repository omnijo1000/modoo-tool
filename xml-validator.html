XML 검증이란?

XML 검증은 XML 문서가 well-formed(올바른 형식) 또는 valid(스키마 규칙 준수) 여부를 확인하는 과정입니다. well-formed XML은 모든 태그가 올바르게 열리고 닫히며, 속성값은 따옴표로 감싸고, 루트 요소가 하나만 존재하는 문서를 의미합니다. 개발 과정에서 XML 설정 파일, API 응답, 사이트맵, RSS 피드 등의 오류를 조기에 발견하고 수정하는 데 XML 검증기가 유용합니다.

자주 묻는 질문

XML 검증이란 무엇인가요?

XML 검증은 XML 문서가 파서가 오류 없이 읽을 수 있는지(well-formed), 나아가 DTD·XSD 스키마에서 정의한 구조 규칙까지 만족하는지(valid) 검사하는 과정입니다. 이 도구는 브라우저 내장 DOMParser를 사용해 well-formed 여부를 즉시 확인합니다.

well-formed XML과 valid XML의 차이는?

well-formed XML은 기본 XML 문법을 따르는 문서입니다. 모든 태그가 올바르게 닫히고, 속성값은 큰따옴표 또는 작은따옴표로 감싸지고, 루트 요소가 정확히 하나 존재해야 합니다. valid XML은 여기에 더해 DTD 또는 XSD 스키마에 정의된 요소·속성 구조까지 준수합니다.

XSD 스키마란 무엇인가요?

XSD(XML Schema Definition)는 XML 문서의 구조, 데이터 타입, 요소 순서, 속성 제약 등을 정의하는 스키마 언어입니다. DTD보다 표현력이 강하며, XML 형식으로 작성됩니다. 예를 들어 sitemap.xml은 sitemaps.org에서 정의한 XSD를 따릅니다.

XML 네임스페이스란 무엇인가요?

XML 네임스페이스는 서로 다른 XML 어휘의 요소 이름 충돌을 방지하는 메커니즘입니다. xmlns 속성으로 선언하며, xmlns:prefix 형태로 접두사를 부여할 수 있습니다. 예: <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">

DTD란 무엇인가요?

DTD(Document Type Definition)는 XML 문서의 구조를 정의하는 초기 스키마 언어입니다. <!DOCTYPE> 선언으로 참조하거나 인라인으로 작성합니다. 현재는 표현력이 풍부한 XSD나 RelaxNG로 대체되는 추세이지만, HTML과 일부 레거시 시스템에서 여전히 사용됩니다.

CDATA 섹션은 언제 사용하나요?

CDATA(<![CDATA[ ... ]]>) 섹션은 <, >, & 등 XML 특수 문자를 이스케이프 없이 그대로 포함해야 할 때 사용합니다. JavaScript 코드나 HTML 조각처럼 특수 문자가 많은 내용을 XML에 포함할 때 유용합니다.

XML 파서와 HTML 파서의 차이는?

XML 파서는 strict 모드로 동작해 문법 오류가 있으면 즉시 중단합니다. 반면 HTML 파서는 오류를 관대하게 처리해 잘못된 마크업도 최대한 렌더링합니다. 따라서 XHTML은 XML 파서로 처리되므로 완벽한 well-formed 문서여야 합니다.

인코딩 선언은 왜 중요한가요?

<?xml version="1.0" encoding="UTF-8"?> 선언은 XML 파서에게 파일의 문자 인코딩을 알려줍니다. 선언과 실제 인코딩이 다르면 파싱 오류나 문자 깨짐이 발생합니다. UTF-8은 한글·한자·일본어 등 다국어를 지원하므로 가장 널리 권장됩니다. BOM(Byte Order Mark)이 있는 UTF-8 파일은 일부 파서에서 오류를 일으킬 수 있으므로 BOM 없는 UTF-8을 사용하세요.
