JSON Schema 검증기

JSON 데이터를 JSON Schema에 따라 검증합니다. 타입 불일치, 필수 필드 누락, 최소/최대 값 위반, 패턴 불일치 등의 오류를 즉시 감지합니다.

자주 묻는 질문

JSON Schema 검증 시 자주 나타나는 오류는?

타입 불일치(type mismatch), 필수 필드 누락(required), 값 범위 초과(minimum/maximum), 패턴 불일치(pattern)가 가장 흔합니다.

required 필드를 모두 검사하나요?

예, required 배열에 정의된 모든 필드의 존재 여부를 검사합니다.

중첩 객체도 검증되나요?

예, 재귀적으로 중첩 객체와 배열 아이템도 검증합니다.

additionalProperties 검증도 지원하나요?

예, additionalProperties: false인 경우 정의되지 않은 필드를 오류로 표시합니다.

AJV와 동일한 결과를 보장하나요?

기본적인 검증은 동일하나, 일부 고급 키워드(anyOf 등)는 지원이 제한될 수 있습니다.

배열 아이템 검증도 지원하나요?

예, items 스키마에 정의된 규칙으로 각 배열 요소를 검증합니다.

오류 경로는 어떻게 표시되나요?

점 표기법(예: user.address.city)으로 오류 발생 위치를 표시합니다.

JSON Schema 버전 차이가 있나요?

이 도구는 Draft-07 기준으로 검증합니다. 버전별 키워드 차이가 있을 수 있습니다.
