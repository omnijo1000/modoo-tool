SQL 유효성 검사란?

SQL 쿼리의 기본 구문 오류를 클라이언트 측에서 검사합니다. 괄호 불일치, 인용부호 미닫힘, 필수 키워드 누락 등을 감지합니다. 또한 SQL 포맷팅 기능으로 쿼리에 들여쓰기와 줄바꿈을 추가해 가독성을 높입니다.

자주 묻는 질문

어떤 SQL 방언을 지원하나요?

표준 SQL 문법을 기준으로 합니다. MySQL, PostgreSQL, SQLite 등 주요 RDBMS의 기본 구문을 지원합니다.

포맷팅이 쿼리 의미를 바꾸나요?

아니요. 포맷팅은 공백과 줄바꿈만 수정하며 쿼리 논리는 변경하지 않습니다.

서버에 SQL이 전송되나요?

아니요. 모든 처리는 브라우저 내에서 이루어집니다. SQL 쿼리는 서버로 전송되지 않습니다.

저장 프로시저도 검사되나요?

CREATE PROCEDURE 구문을 포함한 기본 DDL 검사를 지원합니다.

괄호 균형 검사란?

열린 괄호와 닫힌 괄호의 수가 일치하는지 확인합니다. 누락 시 오류 메시지를 표시합니다.

긴 쿼리도 처리되나요?

브라우저 메모리 범위 내에서 모든 길이의 쿼리를 처리합니다.

키워드 대문자화가 포함되나요?

포맷 결과에서 SQL 키워드는 자동으로 대문자로 변환됩니다.

여러 쿼리(세미콜론 구분)도 처리되나요?

세미콜론으로 구분된 여러 SQL 구문을 개별 포맷합니다.
