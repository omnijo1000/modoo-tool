API 테스터란?

브라우저에서 직접 HTTP API 요청을 테스트합니다. GET, POST, PUT, PATCH, DELETE 메서드와 커스텀 헤더, JSON 본문을 지원합니다. 응답 상태 코드, 응답 헤더, 바디를 즉시 확인하세요. 단, CORS 정책으로 일부 API는 브라우저에서 직접 호출되지 않을 수 있습니다.

자주 묻는 질문

CORS 오류가 발생하는 이유는?

브라우저는 다른 도메인의 API를 호출할 때 서버의 CORS 허용 헤더를 확인합니다. 서버에서 Access-Control-Allow-Origin 헤더를 설정해야 합니다.

인증이 필요한 API도 테스트할 수 있나요?

Authorization 헤더에 Bearer Token 또는 API 키를 추가해 테스트할 수 있습니다.

응답 JSON이 자동으로 포맷되나요?

응답 Content-Type이 application/json이면 자동으로 들여쓰기 처리합니다.

HTTPS API도 테스트할 수 있나요?

네. HTTPS URL을 입력하면 바로 테스트 가능합니다.

전송된 데이터는 안전한가요?

요청은 브라우저에서 직접 API 서버로 전달됩니다. 이 사이트의 서버를 경유하지 않습니다.

요청 시간 초과가 있나요?

브라우저 기본 타임아웃이 적용됩니다. 응답이 오래 걸리면 브라우저가 자동으로 중단합니다.

응답 헤더를 모두 볼 수 있나요?

CORS 정책상 일부 응답 헤더는 브라우저에서 노출되지 않을 수 있습니다.

POST 본문 형식은?

JSON 형식을 사용하세요. Content-Type: application/json 헤더를 함께 설정하는 것을 권장합니다.
