GraphQL 쿼리 빌더란?

GraphQL의 query, mutation, subscription을 시각적 UI로 빠르게 작성합니다. 루트 필드·인수·반환 필드를 입력하면 올바른 GraphQL 문법의 쿼리를 즉시 생성합니다. GraphQL 문법에 익숙하지 않은 개발자가 쉽게 시작하거나 반복 작업을 자동화할 때 활용하세요.

자주 묻는 질문

GraphQL query vs mutation 차이는?

query는 데이터 읽기(GET), mutation은 데이터 쓰기(CREATE/UPDATE/DELETE), subscription은 실시간 이벤트 구독에 사용됩니다.

변수($variable)를 사용하려면?

인수 값에 $변수명 형식으로 입력하면 쿼리 헤더에 변수 선언이 추가됩니다. 예: id: $userId → query ($userId: ID!).

중첩 필드는 어떻게 추가하나요?

이 빌더는 단일 레벨 필드를 지원합니다. 중첩 구조는 생성된 쿼리를 직접 편집하거나 GraphQL Formatter를 사용하세요.

인수 타입을 지정할 수 있나요?

인수 값에 "string", 1(숫자), true(불리언) 형식으로 입력하면 됩니다. 변수 사용 시 타입은 변수 선언부에서 지정하세요.

GraphQL 인트로스펙션이란?

GraphQL 서버에 스키마 정보를 쿼리하는 기능입니다. __schema, __type 등 내장 타입으로 서버의 지원 쿼리·타입을 조회할 수 있습니다.

프래그먼트는 지원하나요?

이 빌더는 기본 쿼리 생성에 집중합니다. 프래그먼트(fragment)는 생성된 쿼리를 직접 편집해 추가하세요.

GraphQL 엔드포인트는?

대부분의 GraphQL API는 단일 엔드포인트(예: /graphql)를 사용합니다. POST 요청으로 {query, variables}를 JSON body로 전송합니다.

REST API와 차이점은?

GraphQL은 단일 엔드포인트에서 클라이언트가 필요한 필드만 지정해 요청합니다. Over-fetching/Under-fetching 문제를 해결하고 타입 시스템을 통해 API 계약을 명확히 합니다.
