Base64 이미지 변환기란?

이미지 파일을 Base64 Data URL로 인코딩하거나, Base64 문자열을 원본 이미지로 디코딩합니다. 인코딩된 이미지는 HTML <img src="data:image/..."> 또는 CSS background-image에 직접 삽입할 수 있어 별도 파일 없이 단일 HTML 파일에 이미지를 포함할 때 유용합니다. PNG·JPEG·GIF·WebP·SVG·BMP를 지원하며 모든 처리는 브라우저에서 이루어집니다.

자주 묻는 질문 (FAQ)

Base64 이미지는 언제 사용하나요?

이메일 HTML 본문에 이미지 삽입, 단일 HTML 파일로 배포할 때, CDN 없이 소규모 아이콘을 CSS에 포함할 때 유용합니다. 단, 파일 크기가 원본보다 약 33% 커집니다.

Data URL과 Raw Base64의 차이는?

Data URL은 data:image/png;base64,... 형식으로 브라우저가 바로 이미지로 해석할 수 있습니다. Raw Base64는 순수 인코딩 문자열로 API 전송이나 서버 처리 시 사용합니다.

용량 제한이 있나요?

브라우저 메모리 한도 내에서 처리 가능하지만, 대용량 이미지(수 MB 이상)는 Base64 인코딩 시 페이지 성능에 영향을 줄 수 있습니다. 큰 이미지는 별도 파일로 호스팅하는 것을 권장합니다.

SVG도 Base64로 변환할 수 있나요?

네. SVG는 텍스트 기반이므로 Base64 외에도 URL 인코딩(data:image/svg+xml,...)으로 삽입할 수 있습니다. 이 계산기는 Base64 변환을 지원합니다.

보안 문제는 없나요?

모든 처리가 브라우저 내에서만 이루어지며 서버로 이미지가 전송되지 않습니다. 민감한 이미지도 안전하게 변환할 수 있습니다.

Base64 이미지를 CSS background-image로 사용할 수 있나요?

가능합니다. url(data:image/png;base64,...) 형태로 CSS에 직접 삽입할 수 있습니다. 다만 CSS 파일 크기가 커지므로 아이콘 등 소형 이미지에만 권장합니다.

Base64로 인코딩하면 파일 크기가 얼마나 커지나요?

Base64는 원본 바이너리 대비 약 33% 크기가 늘어납니다. 1MB 이미지를 Base64로 변환하면 약 1.33MB의 문자열이 됩니다.

Base64 이미지가 SEO에 불리한가요?

검색 엔진이 Base64 이미지를 독립적인 이미지로 인식하기 어렵습니다. 크롤링과 인덱싱 관점에서는 별도 파일로 제공하는 것이 더 유리합니다.
