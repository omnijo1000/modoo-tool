JSON 검증기란?

JSON 검증기(JSON Validator)는 입력된 문자열이 JSON 표준(RFC 8259)에 맞는 올바른 형식인지 검사하는 도구입니다. API 응답, 설정 파일, 데이터 교환 등 다양한 상황에서 JSON을 직접 편집하다 보면 후행 쉼표, 큰따옴표 누락, 괄호 불일치 같은 오류가 쉽게 발생합니다. 이 도구는 JSON을 붙여넣는 즉시 오류 위치와 메시지를 표시하고, 유효한 경우 키 수·중첩 깊이·문자 수 통계를 함께 제공합니다. 모든 처리는 브라우저 내에서만 이루어지므로 API 키나 인증 토큰이 포함된 민감한 데이터도 안전하게 검증할 수 있습니다.

자주 묻는 질문

JSON 유효성 검사란 무엇인가요?

JSON 유효성 검사는 입력 문자열이 RFC 8259 표준에 정의된 JSON 문법을 준수하는지 확인하는 과정입니다. 파서가 입력을 분석하여 큰따옴표 누락, 후행 쉼표, 중괄호 불일치 등의 오류를 보고합니다.

자주 발생하는 JSON 오류는 무엇인가요?

가장 흔한 오류는 배열이나 객체 마지막 항목 뒤의 후행 쉼표, 키나 문자열 값에 사용된 작은따옴표, 닫는 괄호 누락, 문자열 내 이스케이프되지 않은 특수문자입니다.

JSON 검증기와 JSON 포맷터의 차이는 무엇인가요?

JSON 포맷터는 JSON에 들여쓰기와 줄바꿈을 추가해 가독성을 높입니다. JSON 검증기는 입력을 수정하지 않고 오류 위치만 정확히 알려줍니다. 두 기능을 모두 원하면 JSON 포맷터를 사용하세요.

RFC 8259란 무엇인가요?

RFC 8259는 JSON 데이터 교환 형식을 정의하는 인터넷 표준입니다. 모든 키는 큰따옴표로 감싼 문자열이어야 하며, 후행 쉼표·주석·undefined 값은 허용되지 않습니다.

JSON 검증은 언제 사용해야 하나요?

API에 데이터를 전송하기 전, 실패한 네트워크 요청을 디버깅할 때, 설정 파일을 직접 편집한 후, 외부 소스에서 받은 데이터를 통합할 때 JSON을 검증하세요.

JSON에서 주석을 사용할 수 없는 이유는 무엇인가요?

JSON 사양은 파싱을 단순하고 결정적으로 유지하기 위해 의도적으로 주석을 제외했습니다. 주석이 필요하다면 JSON의 상위 집합인 JSONC 또는 JSON5를 사용하세요.

JSON에서 숫자 정밀도 문제가 있나요?

JSON 숫자는 IEEE 754 배정밀도 부동소수점으로 파싱됩니다. 2^53보다 큰 정수는 정밀도가 손실됩니다. 대형 ID나 금융 값은 문자열로 표현하는 것이 안전합니다.

여기에 JSON을 붙여넣어도 안전한가요?

네. 모든 검증은 JavaScript를 사용하여 브라우저 내에서만 실행됩니다. 데이터는 서버로 전송되지 않으므로 API 키, 토큰, 기타 민감한 JSON도 완전히 안전합니다.

모든 처리는 브라우저에서만 이루어집니다 · 서버로 전송되지 않습니다
