텍스트 케이스 변환기란?

텍스트를 다양한 케이스로 즉시 변환합니다. 개발자에게 유용한 camelCase·PascalCase·snake_case·kebab-case·CONSTANT_CASE부터 일반 UPPERCASE·lowercase·Title Case·Sentence case까지 한 번에 변환할 수 있습니다. 변수명·함수명·DB 컬럼명 규칙을 맞출 때 활용하세요.

자주 묻는 질문 (FAQ)

camelCase와 PascalCase의 차이는?

camelCase는 첫 단어 소문자, 이후 단어 첫 글자 대문자입니다(예: myVariableName). PascalCase(UpperCamelCase)는 모든 단어 첫 글자 대문자입니다(예: MyVariableName). JavaScript 변수·함수는 camelCase, 클래스·컴포넌트는 PascalCase가 일반적입니다.

snake_case와 kebab-case는 언제 쓰나요?

snake_case는 Python 변수·함수명, DB 컬럼명에 주로 쓰입니다. kebab-case는 URL 슬러그, CSS 클래스명, HTML 속성에 사용됩니다.

한국어가 포함된 텍스트도 변환되나요?

한국어·한자·이모지 등 비ASCII 문자는 대소문자 변환 대상이 아니므로 영문 부분만 변환됩니다. snake_case·kebab-case 변환 시 공백을 기준으로 분리합니다.

Title Case 규칙은 어떻게 적용되나요?

이 도구는 모든 단어의 첫 글자를 대문자로 변환합니다(a, the, of 등 관사 예외 없음). 언어별 관사 규칙이 필요한 경우 직접 수정하세요.

변환 결과를 바로 복사할 수 있나요?

네. 각 결과 항목을 클릭하면 클립보드에 바로 복사됩니다.

SCREAMING_SNAKE_CASE는 언제 사용하나요?

주로 상수(constants)를 표현할 때 사용합니다. Python, Java, C에서 변경되지 않는 값(예: MAX_RETRY_COUNT = 3)을 나타낼 때 관례적으로 씁니다. 환경 변수 이름에도 자주 사용됩니다.

한 번에 여러 단어의 표기법을 변환할 수 있나요?

가능합니다. 입력창에 변환할 텍스트 전체를 붙여넣으면 모든 단어가 선택한 표기법으로 일괄 변환됩니다. 코드 리팩토링이나 변수명 일괄 교체 시 활용할 수 있습니다.

dot.case 표기법은 어떤 언어에서 사용되나요?

dot.case는 네임스페이스 구분에 주로 사용됩니다. Java 패키지명(com.example.myapp), JavaScript 모듈 경로, 설정 파일 키(spring.datasource.url)에서 볼 수 있습니다.

CSS text-transform과 이 도구의 대소문자 변환은 어떻게 다른가요?

CSS text-transform: uppercase는 화면에 보이는 표시만 바꿀 뿐 실제 문자 데이터는 그대로입니다. 이 도구는 문자열 자체의 값을 실제로 변환하므로, 변환된 텍스트를 복사해 데이터베이스나 다른 시스템에 그대로 사용할 수 있습니다.

터키어 등 일부 언어에서 대소문자 변환이 다르게 동작할 수 있나요?

네. 터키어는 점 없는 대문자 I(I)와 점 있는 소문자 i(İ/ı)를 구분하는 특수한 규칙이 있어, 표준 toUpperCase()/toLowerCase()로 변환하면 예상과 다른 결과가 나올 수 있습니다. 이런 로케일 특수 규칙이 중요한 경우 별도 처리가 필요합니다.
