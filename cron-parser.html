Cron 표현식 파싱이란?

Cron 표현식은 리눅스/유닉스 시스템에서 정기 작업을 자동 실행하기 위한 문자열입니다. 표준 5개 필드(분·시·일·월·요일)로 구성되며, AWS EventBridge, GitHub Actions, Spring @Scheduled 등 다양한 환경에서 사용됩니다. 6번째 필드(초)를 추가하는 확장 형식도 있습니다.

이 도구는 cron 표현식을 파싱하여 인간이 이해할 수 있는 언어로 설명하고, 순수 자바스크립트로 다음 10회 실행 시간을 계산합니다. 외부 라이브러리를 전혀 사용하지 않아 브라우저에서 완전히 동작합니다.

Cron이란 무엇인가요?

Cron은 유닉스 계열 운영 체제에서 정해진 시간에 명령이나 스크립트를 자동으로 실행하는 데몬입니다. crontab 파일에 "언제 무엇을 실행할지"를 cron 표현식으로 정의합니다.

5필드 cron과 6필드 cron의 차이는?

표준 cron은 분·시·일·월·요일의 5개 필드를 씁니다. Spring @Scheduled, AWS EventBridge는 맨 앞에 초(0–59) 필드를 추가해 6개 필드를 사용합니다. 이 파서는 두 형식을 모두 지원합니다.

특수 문자 * / - , 는 각각 무슨 의미인가요?

* 는 모든 값, / 는 간격(*/5 = 5마다), - 는 범위(1-5 = 1부터 5까지), , 는 목록(1,3,5 = 1, 3, 5 각각)을 의미합니다.

자주 쓰는 cron 패턴에는 어떤 것이 있나요?

매분: * * * * * | 매시 정각: 0 * * * * | 매일 자정: 0 0 * * * | 5분마다: */5 * * * * | 평일 오전 9시: 0 9 * * 1-5 | 매월 1일 자정: 0 0 1 * *

Cron에서 타임존 문제는 어떻게 처리하나요?

표준 crontab은 서버의 시스템 타임존을 따릅니다. AWS EventBridge는 UTC 기준이지만 스케줄 표현식에 타임존을 지정할 수 있고, GitHub Actions는 UTC를 기본으로 합니다. 항상 사용 환경의 타임존을 확인하세요.

Cron과 crontab의 차이는?

Cron은 주기적 작업을 실행하는 데몬(백그라운드 프로세스)이고, Crontab은 실행할 작업과 시간을 정의한 설정 파일입니다. crontab -e로 편집합니다.

Cron의 최소 실행 주기는?

표준 cron의 최소 단위는 1분(* * * * *)입니다. 1분 미만의 주기가 필요하면 systemd 타이머, 작업 큐, 또는 cron 내에서 sleep 루프를 사용합니다.

DOM과 DOW를 모두 지정하면 어떻게 되나요?

두 필드 모두 와일드카드(*)가 아닐 때, Vixie cron을 비롯한 대부분의 구현체는 둘 중 하나만 일치해도 실행합니다(OR 조건). 예: 0 0 15 * 1 은 매월 15일이거나 매주 월요일 자정에 실행됩니다.

L과 W 특수문자도 지원되나요?

L(마지막 날)과 W(가장 가까운 평일)는 Quartz Scheduler와 Spring @Scheduled의 확장 문자로, 표준 Linux crontab에서는 지원되지 않습니다. AWS EventBridge는 별도의 특수문자 체계를 가지므로 사용 전 대상 플랫폼의 cron 방언을 확인하세요.
