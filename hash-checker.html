해시 체커란?

텍스트의 SHA-256, SHA-384, SHA-512, SHA-1 해시를 생성하고, 알려진 해시값과 비교 검증합니다. 파일 무결성 확인, 비밀번호 저장 전 해싱, 데이터 무결성 검증 등에 활용합니다.

자주 묻는 질문

SHA-256과 MD5의 차이는?

MD5는 128비트로 충돌 취약점이 발견되어 보안 목적으로는 사용하지 않는 것이 좋습니다. SHA-256은 256비트로 현재 안전한 해시 알고리즘입니다.

해시는 왜 단방향인가요?

해시 함수는 임의 크기의 입력을 고정 크기 출력으로 변환합니다. 역함수가 존재하지 않도록 설계되어 있습니다.

동일한 텍스트는 항상 같은 해시를 생성하나요?

네. 결정론적 함수로, 같은 입력은 항상 같은 해시를 생성합니다.

파일 해시 검증이란?

다운로드한 파일이 손상되지 않았는지 확인하기 위해 제공자의 해시값과 비교합니다.

서버로 텍스트가 전송되나요?

아니요. Web Crypto API를 사용해 브라우저에서만 처리됩니다.

SHA-1은 안전한가요?

SHA-1은 충돌 공격이 발견되어 보안 목적으로는 권장하지 않습니다. SHA-256 이상을 사용하세요.

해시는 암호화인가요?

아니요. 해시는 단방향 함수로 암호화와 다릅니다. 암호화는 복호화가 가능하지만 해시는 복원 불가합니다.

비밀번호에 해시를 사용해도 되나요?

SHA 계열 단순 해시는 비밀번호 저장에 부적합합니다. bcrypt, Argon2, PBKDF2 같은 키 스트레칭 알고리즘을 사용하세요.
