User-Agent(UA)란?

User-Agent는 브라우저나 크롤러 봇이 HTTP 요청 시 서버에 자신을 식별하는 문자열입니다. 브라우저 종류, 버전, 운영체제, 렌더링 엔진 정보를 포함합니다. 서버는 UA를 통해 모바일/데스크탑 분기, 봇 탐지, 통계 수집 등을 처리합니다.

자주 묻는 질문

내 User-Agent를 어떻게 확인하나요?

브라우저 개발자 도구 콘솔에서 navigator.userAgent를 입력하면 확인할 수 있습니다. 이 페이지의 "내 UA 사용" 버튼을 클릭해도 현재 브라우저 UA가 자동으로 입력됩니다.

User-Agent를 변경할 수 있나요?

크롬 개발자 도구 → Network Conditions → User agent 섹션에서 변경 가능합니다. 또는 브라우저 확장 프로그램을 사용하면 웹사이트에 다른 UA로 보이게 할 수 있습니다.

봇과 크롤러를 UA로 탐지하는 방법은?

Googlebot, Bingbot 등은 UA에 bot, crawler, spider 키워드를 포함합니다. if(/bot|crawler|spider/i.test(navigator.userAgent)) 패턴으로 기본적인 감지가 가능합니다.

모바일과 데스크탑을 UA로 구분하는 방법은?

모바일 UA에는 Mobi, Android, iPhone 등의 키워드가 포함됩니다. 그러나 UA 기반 탐지보다 CSS 미디어 쿼리나 window.innerWidth를 사용한 반응형 디자인이 더 정확하고 유지보수가 쉽습니다.

User-Agent Reduction이란?

Chrome 107+부터 Privacy Sandbox 정책으로 UA에서 마이너 버전, OS 버전 등 일부 정보가 제거되었습니다. 상세 정보는 User-Agent Client Hints API(navigator.userAgentData)를 통해 필요시에만 요청할 수 있습니다.

검색 엔진 크롤러 UA 목록은?

주요 크롤러: Googlebot(Mozilla/5.0...Googlebot/2.1), Bingbot(bingbot/2.0), Yandexbot, DuckDuckBot, facebookexternalhit(소셜 공유 미리보기), Twitterbot 등이 있습니다.

AppleWebKit/537.36은 무엇인가요?

WebKit은 Apple이 개발한 브라우저 엔진입니다. Chrome, Edge, Opera 등 Blink 기반 브라우저도 호환성을 위해 537.36을 고정값으로 포함합니다. 실제 렌더링 엔진 버전은 Chrome/xxx 부분을 확인하세요.

UA 파싱 시 주의사항은?

UA는 클라이언트가 임의로 조작할 수 있어 완전히 신뢰해서는 안 됩니다. 서버 사이드 로직이나 보안에 UA를 사용하면 우회될 수 있습니다. 통계, 분기 처리 등 참고용으로만 사용하세요.
