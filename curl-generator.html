cURL 생성기

HTTP 요청 설정 → cURL 명령어 자동 생성

헤더
Bearer Token
요청 본문 (JSON/form-data)
생성된 cURL

cURL 명령 생성기란?

HTTP 요청 파라미터(URL, 메서드, 헤더, 본문)를 입력하면 cURL 명령어를 자동으로 생성합니다. API 테스트, 디버깅, 문서 작성 등에 활용하세요. Bearer Token, Content-Type, Authorization 등 주요 헤더를 손쉽게 추가할 수 있습니다.

자주 묻는 질문

cURL 명령어란 무엇인가요?

cURL은 다양한 프로토콜로 데이터를 전송하는 커맨드라인 도구입니다. API 테스트와 HTTP 요청 자동화에 널리 사용됩니다.

JSON 본문을 포함할 수 있나요?

네. 본문 입력란에 JSON을 붙여넣으면 -d 옵션으로 자동 포함됩니다.

Bearer Token은 어떻게 추가되나요?

Bearer Token 입력란에 토큰을 입력하면 Authorization: Bearer {token} 헤더가 자동 추가됩니다.

-k (insecure) 옵션은 무엇인가요?

SSL/TLS 인증서 검증을 비활성화합니다. 자체 서명된 인증서를 사용하는 개발 환경에서 사용합니다.

-L 옵션은 무엇인가요?

리다이렉트(301, 302 등)를 자동으로 따라갑니다.

헤더를 여러 개 추가할 수 있나요?

+ 추가 버튼을 클릭해 원하는 만큼 헤더를 추가할 수 있습니다.

생성된 명령어를 바로 실행할 수 있나요?

터미널에 복사해 붙여넣기하면 즉시 실행 가능합니다.

Windows에서도 cURL을 사용할 수 있나요?

Windows 10 이상에서는 기본 PowerShell 및 커맨드 프롬프트에서 cURL이 지원됩니다.
