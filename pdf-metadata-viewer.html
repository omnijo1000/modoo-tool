PDF 메타데이터 뷰어

PDF 파일의 제목·저자·주제·키워드·생성일·수정일 등 메타데이터를 확인합니다. 브라우저에서 로컬로 처리됩니다.

자주 묻는 질문

어떤 메타데이터를 확인할 수 있나요?

제목, 저자, 주제, 키워드, 작성/변환 프로그램, 생성일, 수정일, 페이지 수 등입니다.

파일이 서버로 전송되나요?

아니요. 브라우저 내에서만 처리됩니다.

메타데이터 수정도 가능한가요?

이 도구는 확인만 가능합니다. 수정은 지원하지 않습니다.

생성일이 표시되지 않는 경우는?

PDF에 생성일 정보가 없거나 제거된 경우입니다.

암호화된 PDF도 처리되나요?

암호화되지 않은 PDF만 처리됩니다.

PDF 버전 정보도 확인할 수 있나요?

예, PDF 스펙 버전(예: 1.4, 1.7)을 확인할 수 있습니다.

개인정보가 포함될 수 있나요?

저자·생성 프로그램 등에 개인정보가 포함될 수 있습니다. 확인 후 공유 전 삭제를 권장합니다.

메타데이터를 제거하려면?

EXIF 제거기나 PDF 메타데이터 편집 도구를 사용하세요.

각 메타데이터 필드는 정확히 무엇을 의미하나요?

Title=문서 제목(파일명과 무관하게 별도 설정 가능), Author=작성자명, Subject=문서 주제 요약, Keywords=검색용 키워드 목록, Creator=원본 문서를 만든 애플리케이션(예: Microsoft Word, HWP), Producer=PDF로 변환·생성한 소프트웨어(예: Adobe PDF Library, macOS의 Quartz), CreationDate=최초 생성 시각, ModDate=마지막 수정 시각입니다. Creator와 Producer의 차이: Word에서 작성 후(Creator: Microsoft Word) PDF로 내보내면(Producer: Microsoft: Print To PDF 등) 두 값이 서로 다른 프로그램을 가리키는 경우가 많습니다.

이력서·계약서 제출 전 메타데이터 유출 사고 사례는?

실제로 발생하는 사고: ① 이전 지원 회사명이나 이전 프로젝트명이 Title·Subject 필드에 그대로 남아있는 채로 다른 회사에 이력서를 제출 ② 워드 문서의 "변경 내용 추적(Track Changes)" 작성자 정보가 Author 필드나 문서 내부에 남아 실제 작성자(대필 여부 등)가 드러나는 경우 ③ 회사 내부 템플릿을 재사용하면서 Producer 필드에 사내 시스템명이 노출되어 조직 정보가 간접적으로 드러나는 경우. 제출 전 이 도구로 반드시 확인하세요.

XMP 메타데이터와 기존 Info Dictionary는 어떻게 다른가요?

PDF는 메타데이터를 두 가지 방식으로 저장할 수 있습니다. Info Dictionary(전통 방식)는 Title/Author 등 단순 키-값 쌍을 PDF 트레일러에 직접 저장하는 오래된 구조입니다. XMP(Extensible Metadata Platform)는 Adobe가 만든 XML 기반 메타데이터로, Info Dictionary보다 훨씬 풍부한 정보(저작권, 커스텀 스키마, 편집 이력)를 저장할 수 있으며 최신 PDF는 두 가지를 동시에 포함하기도 합니다. 이 도구는 주로 Info Dictionary 필드를 표시합니다.
