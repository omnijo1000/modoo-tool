암호문

텍스트 암호화·복호화 도구

텍스트를 AES-256-GCM, Caesar, ROT13, Vigenere 방식으로 암호화하거나, 같은 페이지의 '복호화' 탭에서 다시 원문으로 되돌립니다. AES-256-GCM은 Web Crypto API를 사용하여 브라우저에서만 처리하므로 서버로 데이터가 전송되지 않습니다. Caesar와 ROT13은 교육용 고전 암호입니다.

자주 묻는 질문

AES-256-GCM이란?

현재 가장 안전한 대칭키 암호화 알고리즘 중 하나입니다. 256비트 키와 인증 태그를 사용합니다.

내 데이터가 서버로 전송되나요?

아니요. 모든 처리는 브라우저에서만 이루어집니다. 외부로 데이터가 전송되지 않습니다.

Caesar cipher란?

알파벳을 N글자씩 이동시키는 고전 치환 암호입니다. 보안성이 낮아 교육용으로만 사용합니다.

ROT13이란?

Caesar cipher의 특수 케이스로 13글자씩 이동합니다. 같은 방법으로 복호화도 됩니다.

Vigenere cipher란?

키 단어를 반복하여 각 문자마다 다른 이동 값을 적용하는 다중 치환 암호입니다.

AES 암호화 결과를 복호화하려면?

이 페이지 상단의 '복호화' 탭으로 전환하여, 암호화할 때 사용한 것과 동일한 비밀번호와 AES-256-GCM 방식을 선택하면 복호화됩니다.

비밀번호를 잃어버리면?

AES 암호화의 경우 비밀번호 없이는 복호화가 불가능합니다.

한국어도 암호화할 수 있나요?

AES와 ROT13(비알파벳 보존)은 모든 언어 텍스트를 처리합니다. Caesar/Vigenere는 영문자만 변환합니다.

이 페이지에서 바로 복호화할 수 있나요?

네. 상단의 '복호화' 탭을 클릭하면 같은 페이지에서 암호문을 붙여넣고 복호화할 수 있습니다. 별도의 페이지로 이동할 필요가 없습니다.
